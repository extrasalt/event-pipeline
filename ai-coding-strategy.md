# Draft
Baseline generation should be done by deterministic tools like `vite create`. Ensure that the agent uses libraries from npm or shadcn componenents where appropriate. Do not have it generate code that's available off the shelf. If a popular library exists for the purpose, we should use it rather than maintain generated code. 

Unit tests can be generated by LLMs with little supervision as long as missing tests are added after the fact. Frontend components that pertain to the UX design, especially the tailwind classes can be generated without supervision as any deficiencies would become apparent when the UI is rendered. 

YAML configs for auxillary tools like grafana can also be safely generated. 

Code quality is maintained by making sure abstractions are created, as that's a common AI miss. if the code is properly layered, any future changes would not affect the deepest parts of the codebase and the changes would be localized. I like to call this "cake code" in contrast to "sphegetti code".

Creating the wrong kind of abstractions prematurely will bite us in the future, so asking AI to aggressively create abstractions too has downsides.
